## 💻 軟體人格 — 程式碼品質檢核

### 2.1 TypeScript 型別正確性

| 檢查項目 | 位置 | 結果 | 說明 |
|---------|:----:|:----:|------|
| GameState 型別 | store/gameStore.ts | ✅ | 完整定義，與 types.ts 一致 |
| Enemy 型別 | types.ts | ⚠️ | `x/y` 座標缺失 — `MapScreen.tsx:57` 嘗試找 `enemy.x/y` 但類型無此欄位 |
| Boss 型別 | types.ts | ⚠️ | `Boss` 類型定義了但**從未被使用**，遊戲用 `Enemy` 替代 |
| Upgrade 型別 | types.ts | ⚠️ | `Upgrade`、`FloorConfig` 類型定義了但未使用 |
| resolveAttack return | gameStore.ts:25-37 | ✅ | 明確型別宣告 |
| attack return 型別 | gameStore.ts:247 | ✅ | `ReturnType<typeof resolveAttack>` 良好用法 |

**發現問題**：
1. **❌ P1** `敵人物件缺少座標` — `MapScreen.tsx:57` 中 `enemies.find(e => e.x === x && e.y === y)` 永遠回傳 undefined，因為 `Enemy` 類型無 `x/y` 欄位。敵人在地圖上無法正確定位
2. **⚠️ P2** `3 個未使用的類型` — Boss、Upgrade、FloorConfig 定義了但未被實作使用

### 2.2 Zustand State 管理

| 檢查項目 | 結果 | 說明 |
|---------|:----:|:-----|
| Store 結構清晰度 | ✅ | 單一 Store 模式，分工明確 |
| Monolithic 問題 | ⚠️ | 639 行單一檔案，含地城生成、戰鬥、道具、商店、存檔 |
| state selector 效率 | ✅ | Screens 使用 selector `s => s.field` 避免不必要 re-render |
| Stale closure | ⚠️ | `attack()` 中 `currentEnemy` closure 已部分修復（capture enemyName） |
| Fire-and-forget Promise | ⚠️ | `saveGame()` 在 `attack()`、`useItem()`、`buyItem()` 中 fire-and-forget，無 error handling |

**發現問題**：
3. **⚠️ P2** `attack()` 中 `currentEnemy.currentHp` 在 `set()` 前可能 stale — 使用 `const enemyName = currentEnemy.name` 捕獲但 HP 未捕獲
4. **⚠️ P2** `saveGame()` Promise 未處理 — 3 處 fire-and-forget 呼叫，存檔失敗時無回退機制

### 2.3 邊界情況處理

| 邊界情況 | 檢查結果 | 說明 |
|---------|:--------:|------|
| 空道具背包使用 | ⚠️ | `useItem()` 中 `player.inv.find(i => i.id === itemId)` 若未找到直接 return，無提示 |
| 滿血用藥水 | ⚠️ | `useItem()` 無檢查 HP 是否已滿，玩家可浪費藥水 |
| 死亡後操作 | ✅ | `over: true` 後 `GameOverOverlay` 蓋住所有操作 |
| 金幣不足購買 | ✅ | `buyItem()` 檢查 `player.gold < item.cost` 後 return |
| 空裝備欄卸載 | ✅ | `unequipItem()` 檢查 `if (!item) return` |
| 武器/護甲類型誤用 | ✅ | `equipItem()` 檢查 `item.type !== 'weapon' && item.type !== 'armor'` |
| 樓層 0 或負數 | ✅ | floor 從 1 起始，`nextFloor` 只 ++ |
| 移動超出地圖邊界 | ✅ | `movePlayer()` 檢查 `ny < 0` 等 |
| 敵人陣列為空時戰鬥 | ✅ | `nearbyEnemies.length === 0` 顯示「安全」資訊 |

**發現問題**：
5. **⚠️ P2** `滿血用藥水無防呆` — 玩家 HP 已滿時仍可使用生命藥水，浪費道具
6. **⚠️ P3** `useItem()` 缺少 movePoints 檢查 — SPEC v3.0 寫明應消耗 2 MOVE，但查看實際程式碼 lines 481-524：

```typescript
// line 481-527 - useItem 已實作 MOVE 消耗！
const mp = get().movePoints;
const newMp = Math.max(0, mp - 2);
set({ movePoints: newMp });
```
實際程式碼**已實作** MOVE 消耗（lines 519-524），但 `USE_ITEM_NO_MOVE_CHECK` 在 SPEC 中標記為已知問題 — 需確認版本。程式碼先於 SPEC 更新。
